# Anti-AI Lint（反 AI 套话静态检查清单）

> 这是 stage 5.5.3 anti-AI lint 的检查规则集。设计上必须能用正则一次扫完整篇 report.md。
> 源自 B3.2-B3.5 隐式偏好，所有规则都来自 Google I/O '26 简报项目反复迭代。

---

## 检查策略

```
report.md → 按规则集扫描 → 输出违规清单（行号 + 规则 + 上下文）→ 必要时 LLM 重写
```

## 规则集（按严重度分级）

### Level 1：硬错误（必须修）

#### L1.1 emoji 黑名单

```python
FORBIDDEN_EMOJI = ["🔥", "🚀", "💡", "⭐", "⭐️", "🎉", "👀", "🤔", "🎯", "✨", "🌟", "⚡", "⚡️", "🎁", "✋", "👋", "💪", "🙌", "💯", "🌈", "🎊"]
```

例外白名单：`📎`（补充信源）、`✅`（表格状态）、`⚠️`（警告引用块）、`📌`（仅用于 `## 📌 整体概要` 标题）

#### L1.2 AI 套话句式

```python
FORBIDDEN_PHRASES = [
    "让我们一起来看看",
    "让我们来看看",
    "让我们看看",
    "值得我们关注的是",
    "不仅仅是",
    "在当今的",
    "这无疑标志着",
    "未来已来",
    "以下是关于",
    "不难看出",
    "众所周知",
    "相信大家都",
    "总而言之",
    "综上所述",
    "由此可见",
    "毋庸置疑",
    "可以预见",
    "众所瞩目",
]
```

#### L1.3 缺失硬约束元素

| 检查 | 规则 |
|---|---|
| 概要 callout | 必须有 `<div class="callout" markdown="1">` |
| 整体概要标题 | 必须有 `## 📌 整体概要` |
| 一点观察 | 必须有 `## 十X、一点观察（独立判断，非发布会原话）` 或类似 |
| 信源说明 | 必须有 `## 信源说明` |
| 透明声明 | `## 信源说明` 节必须含「**官方未公布**」或「**未查到**」 |
| 补充信源最低数量 | `> 📎 **补充信源**` 块 ≥ 8 |
| 章节图覆盖率 | 每个 `## <数字>、` 章节至少 1 张图 |

### Level 2：建议修（可豁免，需 LLM 评估）

#### L2.1 夸张形容词

```python
WARN_ADJECTIVES = ["巨大", "显著", "革命性", "惊人", "震撼", "重磅", "史诗级", "颠覆性", "空前"]
```

豁免条件：上下文是引用演讲者原话（`> "..."` 块内允许）。

#### L2.2 程度副词

```python
WARN_INTENSIFIERS = ["大幅", "极大地", "非常", "十分"]
```

豁免：「明显」可保留（事实性而非夸张）。

#### L2.3 营销词

```python
WARN_MARKETING = ["让人眼前一亮", "令人印象深刻", "不容小觑", "值得期待", "引人注目"]
```

---

### Level 3：风格建议（不强制，仅提示）

#### L3.1 长句（中文 > 60 字）

把长句标出来供 LLM 自动拆短。

#### L3.2 段落过长（> 8 句 / > 400 字）

提示拆分。

#### L3.3 缺数据对比

如果一段含数字但只有单一数字（无 before/after / 倍数），提示补充对比。

#### L3.4 缺表格

如果连续 5 个 bullet 都是平级并列结构，提示改成表格。

## 实现要点

### 1. 引用块豁免

`> "..."` 引用块（演讲者原话）和 `> 📎` 块（补充信源）内的内容**全部豁免** Level 1 / 2 检查（演讲者本人可能用夸张词，不归我们管）。

### 2. 标题豁免

`#` `##` `###` 标题行豁免 L2（标题里偶尔需要副标题修饰）。

### 3. caption 豁免

`![...](path)` 中的 caption 豁免 L1.1 emoji 检查（caption 里有时需要标识符）。

### 4. 章节图覆盖检查

```python
def check_image_coverage(content: str) -> list[str]:
    sections = re.split(r"\n## ", content)[1:]  # 跳过文档头
    missing = []
    for sec in sections:
        title_line = sec.split("\n")[0]
        if not re.search(r"!\[.*?\]\(.*?\)", sec):
            missing.append(title_line)
    return missing
```

如果章节是「信源说明 / 一点观察 / 整体概要」（无图章节），可豁免。
具体规则：只要标题以「一、二、三、四、五、六、七、八、九、十、十一、十二、十三」等中文数字开头的章节，必须有图。

### 5. 补充信源数量检查

```python
def count_citations(content: str) -> int:
    return len(re.findall(r"^>\s*📎\s*\*\*补充信源\*\*", content, re.MULTILINE))
```

阈值：< 8 报错（L1.3）。

## 误报处理

如果 LLM 在某个具体场景下用了禁用词且确实合理（极罕见），允许通过 `<!-- lint-ignore: 巨大 -->` 注释豁免：

```markdown
模型规模有<!-- lint-ignore: 巨大 -->巨大跨越，从 9.7T 到 3,200T。
```

但**强烈不建议**——优先重写句子。

## 与 stage 5 的关系

```
stage 5（draft）
    ↓ 产出 report.md
stage 5.5.3（anti-AI lint）
    ↓ 输出 lint report
若 Level 1 > 0 → 用 lint report 让 LLM 重写问题段落
若 Level 1 = 0 → 通过，进入 stage 6（render HTML）
```

所有 lint 规则都是**宁可误报、不可漏报**——AI 套话比缺一条 emoji 更难纠。
