## 关键长度参考

| 部分 | 行数 / 字数 |
|---|---|
| 整体概要 | 8-12 块 × 3-5 行 ≈ 60-80 行 |
| 主线章节（如 Agent 体系）| 80-120 行（4-6 张图）|
| 重要章节（如 Search / 基建）| 50-80 行（2-4 张图）|
| 次要章节（如 安全 / 科学）| 30-50 行（1-2 张图）|
| 一点观察 | 80-120 行（6-10 个观察）|
| 信源说明 | 30-50 行 |
| **总计** | **600-900 行** |

参考：本项目黄金标准 [io26-keynote-recap.md](../docs/examples/io26-keynote-recap.md) 共 781 行 / 47 张图 / 14 章节。

## 必备元素 checklist

按顺序检查：

- [ ] 文档第一行是 `# <一句精准点题的中文标题>`
- [ ] `<div class="callout" markdown="1">` 包裹整体概要
- [ ] 概要内 `## 📌 整体概要` 标题
- [ ] 概要 8-12 块，每块带 `**核心判断**：`
- [ ] `</div>` 闭合后跟 `---` 分隔
- [ ] 主线章节用 `## 一、` `## 二、` 中文数字
- [ ] 子章节用 `### 1.1` `### 1.2` 阿拉伯数字
- [ ] **每个 ## 章节至少 1 张图**（A8 硬约束）
- [ ] 至少 8 个 `> 📎 **补充信源**：` 块
- [ ] 文末有 `## 十X、一点观察（独立判断，非发布会原话）`
- [ ] 一点观察含 6-10 个独立观察
- [ ] 文末有 `## 信源说明` 节
- [ ] 信源说明含「字幕原话 / 📎 补充信源 / 未查到 / 我的判断」四个子项
