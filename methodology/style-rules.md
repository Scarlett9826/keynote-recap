# 文风规则（反 AI 套话 + 数据驱动叙事）

> 这是 keynote-recap 写作风格的强约束清单。源自 Google I/O '26 简报项目的隐式偏好（B3）。
> 所有 stage 5 prompts 都引用本文。

---

## 1. 反 emoji（B3.2）

### 仅以下两个 emoji 允许

| Emoji | 用途 |
|---|---|
| `📎` | 「补充信源」标记（固定） |
| `✅` | 表格里的「已发布 / 已上线 / 已支持」状态 |

### 禁用清单

❌ 🔥 🚀 💡 ⭐️ 🎉 👀 🤔 🎯 ✨ 🌟 ⚡️ 🎁 ✋ 👋 💪 🙌

理由：emoji 在科技分析文中破坏专业感，且飞书渲染时可能丢失。

## 2. 反夸张词（B3.3）

### 形容词类禁用

| 禁用词 | 替代策略 |
|---|---|
| 巨大 | 给具体数字 + 倍数 |
| 显著 | 给 before / after 对比 |
| 革命性 | 给改变了什么、为什么是结构性 |
| 惊人 | 同上 |
| 震撼 | 删掉，让数据自己说话 |
| 重磅 | 删掉 |
| 史诗级 | 删掉 |
| 颠覆性 | 给具体改变的对象 |
| 空前 | 给历史对比 |

### 程度副词类禁用

| 禁用词 | 替代 |
|---|---|
| 大幅 | 给数字（如「41%」） |
| 极大 | 同上 |
| 非常 | 删掉 |
| 十分 | 删掉 |

特殊情况：「明显」可以保留（事实性描述，非夸张）。

### 营销词类禁用

❌ 让人眼前一亮、令人印象深刻、不容小觑、值得期待、引人注目

## 3. 反 AI 套话（B3.4）

### 禁用句式清单

❌ 「让我们一起来看看……」
❌ 「值得我们关注的是……」
❌ 「不仅仅是……更是……」
❌ 「在当今的……时代」
❌ 「这无疑标志着……」
❌ 「未来已来……」
❌ 「以下是关于 X 的……」
❌ 「不难看出……」
❌ 「众所周知……」
❌ 「相信大家都……」
❌ 「总而言之……」（结尾处）
❌ 「综上所述……」（结尾处）
❌ 「由此可见……」

### 替代策略

| 套话 | 替代 |
|---|---|
| 让我们看看 X 的核心能力 | X 三类核心能力（直接列表） |
| 值得关注的是 Y | 给 Y 是什么 |
| 不仅仅是 A，更是 B | A 和 B 是同一件事的两面（如果是真的） |
| 在 AI 时代 | 删掉，直接说事实 |
| 总而言之 | 删掉，让最后一段就是结论 |

## 4. 数据驱动叙事（B3.1）

### 「时间→数据」三段对比序列（必备）

```markdown
| 维度 | 旧值 | 中间值 | 新值 |
|---|---|---|---|
| Capex | $31B（2022） | — | $180-190B（2026） |
| Token | 9.7T（2024.5） | ~480T（2025.5） | 3,200T（2026.5） |
```

### 数据呈现规则

1. **绝对值 + 倍数**至少给两个：
   - ❌「Capex 从 $31B 涨到 $180B」
   - ✅「Capex $31B（2022）→ $180-190B（2026），约 6×」

2. **同比 / 环比**优先于纯绝对值：
   - ❌「月活 9 亿」
   - ✅「Gemini app MAU：去年 4 亿 → 今天 9 亿+（翻倍以上），日均请求量同比 7 倍+」

3. **百分比 vs 倍数**：
   - 倍数 < 2× 用百分比（如 41%）
   - 倍数 ≥ 2× 用「N×」（如 4×、12×、6×）

4. **量级跨越用科学计数 + 对比**：
   - ✅「9.7T → 480T → 3200T」（万亿级跳变）

### 单位规范

| 类型 | 写法 |
|---|---|
| 美元金额 | `$31B` / `$1,000` / `$200/月` |
| 用户量 | `9 亿+` / `13 款产品 MAU > 10 亿` |
| Token 量 | `3.2T tokens/天` |
| 百分比 | `83.6%`（含小数点保留 1 位） |
| 倍数 | `4×` / `12×` / `约 6×` |
| 时间 | `12 小时` / `5 分钟快充一天` |

## 5. 表格优先并列信息（B3.6）

### 必须用表格的场景

✅ 发布清单 / 时间线
✅ 产品对比 / 价格档位
✅ 多产品规格对比
✅ 合作伙伴列表（≥ 4 个）
✅ 上线时间表
✅ 三种 / 四种典型场景（结构化时）
✅ 三道护栏 / 四个层次类信息

### 标准表格模板

**对比表**：
```markdown
| 维度 | 选项 A | 选项 B |
|---|---|---|
| ... | ... | ... |
```

**时间线表**：
```markdown
| 时间 | 事件 |
|---|---|
| 2026-01 | UCP 首发 |
| 2026-05 | I/O 扩展 |
```

**清单表**：
```markdown
| # | 项目 | 状态 |
|---|---|---|
| 1 | ... | 今日 |
```

## 6. 句式偏好

### 短句多于长句

- 单句 ≤ 30 字（中文）
- 长概念拆成多句

**反例**：
```
今年发布主角从单个模型上升到一整套涵盖前端、中台、底层的完整 Agent 栈，包含 Spark / Information Agents / Daily Brief / Halo / UCP / AP2 / Universal Cart / Search 生成式 UI / Antigravity 2.0 / Managed Agents API。
```

**正例**：
```
今年发布主角从单个模型上升到一整套 Agent 栈：
- **前端**：Spark / Information Agents / Daily Brief / Halo
- **中台**：UCP / AP2 / Universal Cart / Search 生成式 UI
- **底层**：Antigravity 2.0 + Managed Agents API
```

### 主动语态优先

- ❌「Spark 被宣布……」
- ✅「Sundar 宣布 Spark……」

### 段落结构

- 每段 2-5 句
- 段首句给结论，后面给证据
- 段间留空行

## 7. 引用块用法

### 演讲者原话用 `> "..."` 块

```markdown
> "AGI is now on the horizon and it will be the most profound and impactful technology ever invented." —— Demis Hassabis
```

### 补充信源用 `> 📎` 块

```markdown
> 📎 **补充信源**：根据 [blog.google](URL)，<内容>
```

### 警告 / 注意用 `> ⚠️` 块

```markdown
> ⚠️ 此时间段未找到合格的 PPT/演示画面
```

### 普通 `>` 引用块仅用于章节定位说明

```markdown
> **定位**：今年 I/O 的发布主角，从单个模型上升到了整套 Agent 栈。
```

## 8. 「核心判断」格式（B3.5）

每个章节末尾或概要每块末尾，可放一个「**核心判断**：…」收尾：

```markdown
**核心判断**：3.5 Flash 是「牺牲知识深度，换取执行能力」——Google 在打吞吐量战争，不是 benchmark 战争
```

### 写法规则

- 必须有「**核心判断**：」前缀
- 一句话（≤ 50 字）
- 必须是独立分析，不是定调的同义重复
- 用「——」分隔上下半句（视情况）
- 倾向于用「X 不是 A，而是 B」「这是 X 的下一次 Y」「谁定义 X 谁握住 Y」等论断式句型

## 9. 章节标题规范

### L1 章节标题

```markdown
## 一、Agent 体系（最大主线）：从「回答问题」到「替你执行」
```

格式：`## <中文数字>、<主题>（<副标题>）：<点题副句>`

### L2 子章节标题

```markdown
### 1.1 Gemini Spark — 24/7 个人 Agent
```

格式：`### <数字>.<数字> <产品名> — <一句定位>`

### 不要做的事

- ❌ 标题里加 emoji
- ❌ 标题里有冒号 + 长句
- ❌ 标题用问句

## 10. 章节按重要性而非时间序（B3.7）

> 来源：msg #18

### 排序原则

```
最重大主线（如 Agent 体系）
    ↓
主力开发者平台（如 Antigravity）
    ↓
模型层
    ↓
垂类产品（Search / 电商 / 创意工具）
    ↓
新形态硬件（眼镜 / 健康）
    ↓
基建底座（TPU / Capex）
    ↓
科学与安全
    ↓
订阅价格
    ↓
交付节奏（含「一点观察」）
```

### 判断重要性的依据

1. 演讲者花了多少时间讲（时长是粗略指标）
2. 是否带来生态级改变（协议层 > 产品层 > 功能层）
3. 是否是行业首发或竞争对决信号
4. 是否是叙事主线（而非配套发布）
