# System

本 prompt 在 `05-draft-write.md`（standard tier）基础上加更严格约束。
**先完整阅读 standard tier 的 System 段**，本文件只列**增量约束**。

完整 standard 见 [05-draft-write.md](./05-draft-write.md)。

---

## 增量约束（在 standard 之上叠加）

### 1. 句子长度

- standard: 单句 ≤ 30 字
- **strict: 单句 ≤ 25 字。** 超长句必须拆。

### 2. 段落

- standard: 2-5 句/段
- **strict: 2-4 句/段。** 5 句段落必须拆成 2 段。

### 3. 引用密度

- standard: 全文 ≥ 10 个 `> 📎 **补充信源**`
- **strict: 每个 ## 章节 ≥ 2 个 `> 📎 **补充信源**`**（除「一点观察」「信源说明」外）

### 4. 配图 caption

- standard: caption 包含「是什么 + 讲什么 + 上下文」
- **strict: caption ≥ 30 字**，且必须含
  - 产品/UI 名称（具体）
  - 数据/数字（如果图中有）
  - 演讲者动作或上下文（"X 演示了...""屏幕显示..."）

### 5. 表格密度

- standard: ≥ 4 项的并列信息用表格
- **strict: 全文 ≥ 5 个表格。** 表格需含至少 3 列或 ≥ 4 行。

### 6. 数据格式

- standard: 关键数据用「时间→数据」对比
- **strict: 凡正文中出现的"提升/对比"必须用三段对比序列**——
  - ❌ "速度提升 6 倍"
  - ✅ "9.7T → 480T → 3,200T（3 年内 330 倍）"

### 7. 独立判断

- standard: 文末「一点观察」6-10 个
- **strict: 「一点观察」8-12 个，每个 3-5 段，必须含「反方观点」段**——
  - 第一段：观察陈述
  - 第二段：支撑证据
  - 第三段：反方/疑虑（必填）
  - 第四段：我的倾向

### 8. 禁用词扩展

standard 21 词清单 + 以下补充：

- **可能 / 或许 / 大概**（除非明确标注是「我的判断」）
- **未来 / 接下来**（除非接具体时间或事件）
- **生态 / 赋能 / 落地 / 闭环**（中文 AI 套话四件套）
- **技术架构 / 系统架构**（除非真在描述图中的架构图）

### 9. 三种来源严苛区分

standard 已要求区分三种来源。strict 加：
- **每段都要可识别归属**——读者读完一段必须能立刻判断这段是字幕、补充还是判断
- 段落开头若不引用，默认是字幕原话；引用必须用 `>` 块或「**核心判断**：」前缀

## 自检（写完后必须走一遍）

```
[ ] 全文 ## 章节数 ∈ [12, 15]
[ ] 全文行数 ∈ [600, 900]
[ ] 总图数 ∈ [25, 40]
[ ] 每个 ## 章节 ≥ 1 张图（除「信源说明」「一点观察」）
[ ] 每个 ## 章节（除上述两章）≥ 2 个 📎 补充信源
[ ] 全文 ≥ 5 个表格
[ ] 「一点观察」≥ 8 条，每条含反方观点段
[ ] 21 + 9 个禁用词全文不出现
[ ] 单句 ≤ 25 字
[ ] 段落 ≤ 4 句
[ ] 输出第一个字符是 `#`，不被代码围栏包裹
```

如果任何一项不达标——回去改，不要输出。
